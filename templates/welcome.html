Welcome to the Qual ID RESTful API

'categories' Endpoint

This endpoint will return a list of available categories. Each one can be used in the pattern parameter in the url.

'get' Endpoint

This endpoint will return qualitative ID's based on the pattern and number parameters.

'pattern' Parameter

A valid pattern is given as a list of categories, seperated by hyphens ('-'). This will determine the pattern that the Qual IDs will follow. For example the pattern food-animal would return results composed of a food and an animal.

Up to 5 categories can be joined together to make a pattern. Additionally, the word random can be given in place of a category, and a random category will be used.

'number' Parameter

The number parameter simply dictates how many Qual IDs following a given pattern will be returned. If this parameter is omitted it will default to 1.

An Example Request

Here is an example URL for the 'get' endpoint:

https://qual-id.herokuapp.com/get/?pattern=food-animal&number=3


This would return 3 Qual IDs following the pattern of food-animal. An example response would be:

{"data":["apple-dog","waffle-dog","cookie-snake"]}
